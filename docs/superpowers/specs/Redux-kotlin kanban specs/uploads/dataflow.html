Design §2 — data flow, effects, undo, bot

How actions travel and where side-effects / history / the fake backend live. Everything is plain actions through one store — components react to state, never to each other.

Middleware pipeline (applyMiddleware order = outer → inner) on the ConcurrentModelStore
dispatch(action) → ① activityLogger
humanize → ActivityModel → ② undo
snapshot board before undoable actions; handle Undo/Redo → ③ effects (coroutine)
Request → launch → Success/Failure → reducer
combineModelReducers (routed on<Action>) → new ModelState

+ granularSubscriptionsEnhancer so columns/badges/detail/feed each re-render off their own slice. Form keystrokes stay in local Compose remember (not the store) — committed via dispatch on submit.

Async fake backend (simulated latency)
FakeBackend = in-memory repo, suspend fns, 300–800ms delay, occasional failure.

Optimistic flow (move a card):
  1. CardMoveRequested → reducer updates board immediately (optimistic)
  2. effects middleware persists via backend; SyncModel.inFlight++
  3. CardMoveSucceeded → clear inflight · or …Failed → revert + toast + Retry

Demonstrates loading/error UI, retries, and Request/Success/Failure action triples — the real-world async shape.

Undo / Redo
UndoModel: past[] / future[] snapshots of the board slice (capped, e.g. 50).

Undoable: move / create / edit / delete card.
Not undoable: nav, filter, loading flags, session, bot presence.

Undo → push present→future, restore past top, re-persist via the same effect path. Redo reverses.

Lives in the undo middleware, so the UI just dispatches Undo/Redo.

Bot collaborator
A coroutine (started by middleware, toggleable) periodically dispatches normal actions — move/add a card — simulating a 2nd user.

Drives "2 online" + activity feed. Proves the UI reacts to externally-originated state changes with zero coupling — components never know the action came from a bot vs you.
Navigation = Redux state (wasm/iOS-safe)
NavModel: route = BoardList | Board(id), openCardId?.
Compose renders from selectorState{ nav.route }; back = dispatch.

Entering a board → inject its models + fire load effect. Leaving → eject (bounded memory, §1). No external nav lib → identical on iOS / Android / Web.
