C — ReduxMart (Storefront + cart)

Product grid, header cart badge, cart drawer, product detail, checkout stepper. Shared state everywhere: the cart badge + "in cart" markers + totals all derive from one cart model. Classic — but maybe too close to the canonical Redux todo/cart demo.

My recommendation: A — TaskFlow. It maximizes redux-kotlin-granular (each column/badge subscribes to just its slice, so you visibly see render-isolation), uses several ModelState models (Board / Filter / Session), the StoreRegistry (multiple boards), and routed reducers — the widest, most honest API coverage. B is the most intuitive "shared state" story; C is the most familiar.
