Screens

All seven TaskFlow screens, drawn from the approved Phase 1 tokens. Each shows a compact (phone) and expanded (web/desktop) layout in real device frames, with numbered redlines mapping every element to its tokens, dp metrics, and bundle data source.

iOS · Android · Browser frames Compact + Expanded Dark on Board & Card detail 48 dp min targets

How to read these

1Numbered pins

Each pin on a mockup maps to a numbered redline on the right with dp, token, and component ref.

iOSAndroidWeb

Frame badges show the platform. The UI is one Compose M3 surface; frames show platform context only.

Platform notes

Each screen ends with iOS / Android / Web adaptation notes — safe areas, predictive-back, hover, breakpoints.

Adaptation, not restyle

Compose Multiplatform renders the same M3 Expressive surface on every target. Platform differences are adaptation: iOS safe-area insets & back-swipe, Android edge-to-edge & predictive-back, Web hover/pointer states & a 1280 dp content cap. Where a layout genuinely changes, it's the window size class (compact/medium/expanded) driving it — not the OS.

Spec · 9

Asset list

All imagery loads through Coil 3 (per spec §6). Sources are free, deterministic, keyless stock services; every asset has a bundled composeResources fallback so the demo runs fully offline and survives a flaky network (which the failure-rate knob simulates).

AssetSource URL patternUsed inOffline fallback
Account avatarsi.pravatar.cc/150?u={accountId}Login, switcher, app bar, profile, assignee, presenceMonogram Avatar (deterministic tonal bg from id) — already the loading/empty state, so no extra asset needed.
Avatar (alt)api.dicebear.com/9.x/glass/svg?seed={id}Bot collaborator, seeded demo usersBundled SVG set in composeResources/files/avatars/.
Card image attachmentspicsum.photos/seed/{cardId}/600/400KanbanCard image, card-detail image attachment3–4 bundled JPEGs in composeResources/files/cards/, picked by hash.

Loading & error policy

Coil states map to UI: loading → skeleton/shimmer placeholder at the image's aspect ratio; error → bundled fallback (never a broken-image glyph); success → cross-fade in (Effects·Default). The same path covers the Settings failure-rate knob.

Why keyless + bundled

pravatar / dicebear / picsum need no API key and give stable, seedable results — ideal for a reproducible sample. Bundling fallbacks keeps the showcase honest on wasmJs (CORS/offline) and in CI screenshot tests, where the network shouldn't be a dependency.
